Dynamic Product Retrieval API Documentation

Endpoint: Products routes

This endpoint allows you to retrieve a list of products based on dynamic filtering criteria using query parameters.

Query Parameters:

Parameter Description Example
featured Filter products by whether they are featured. ?featured=true
company Filter products by the company that produces them. ?company=ExampleCompany
name Search for products with a name that matches a case-insensitive regular expression. ?name=product
sort Sort the results by specific fields. You can specify multiple fields separated by commas. ?sort=price,-rating
fields Select specific fields to include in the results. You can specify multiple fields separated by commas. ?fields=name,price
numericFilters Apply numeric filters to fields like 'price' and 'rating' using comparison operators. Format: field-operator-value. You can specify multiple filters separated by commas. ?numericFilters=price>50,price<100
page Specify the page number for pagination. Default is 1. ?page=2
limit Specify the number of items per page. Default is 10. ?limit=23

Example Request:

        GET /api/v1/products?featured=true&company=ExampleCompany&name=product&sort=price,-rating&fields=name,price&numericFilters=price>50,price<100&page=2&limit=20

Example Response:

        
{
    "products": [
        {
            "name": "Product A",
            "price": 75.99
        },
        {
            "name": "Product B",
            "price": 89.99
